PACE (oder auch Partsche genannt)

Für den ersten der drei Protokollschritte ist entweder die CAN, die eID-PIN, die PUK oder die MRZ notwendig. Für eigene Experimente bietet sich die CAN an, da man mit ihr den geringsten Schaden anrichten kann.

Die eigene CAN muss zunächst im doeac.js Script hinterlegt werden. Das erledigt die Zeile

var can = "488444";

Für PACE wird auch bereits festgelegt, welche Zugriffsrechte der Ausweisinhaber überhaupt zulassen will. Jedes Bit steht hier für den Zugriff auf ein bestimmtes Datenelement. Selbst wenn ein Terminal aufgrund seines Zertifikats mehr Rechte hat, wird der Chip diese Rechte auf den kleinsten gemeinsamen Nenner zusammenstreichen.

Die Rechtebits finden sich im Objekt mit dem Tag '53'.

var chat = new ASN1(0x7F4C, 
    new ASN1(ASN1.OBJECT_IDENTIFIER, new ByteString(PACE.id_IS, OID)),
    new ASN1(0x53, new ByteString("03", HEX))
    );
